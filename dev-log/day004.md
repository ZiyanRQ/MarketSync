# Pumpkin Farm Evolution

## V1 - Brute Force Farming

```python
while True:
    for Scan in range(4 * get_world_size()):
        for i in range(get_world_size()):

            if get_ground_type() != Grounds.Soil:
                till()

            plant(Entities.Pumpkin)

            if get_entity_type() == None:
                plant(Entities.Pumpkin)

            is_dead_pumpkin = get_entity_type() == Entities.Dead_Pumpkin

            if is_dead_pumpkin:
                plant(Entities.Pumpkin)

            move(North)
        move(East)
```

### What I Learned
- Brute force works but is inefficient  
- No real state awareness  
- Code becomes repetitive quickly

## V2 - First Logical Harvest System

```python
while True:
    for y in range(get_world_size()):
        for x in range(get_world_size()):

            if get_ground_type() != Grounds.Soil:
                till()

            if can_harvest():
                harvest()
                plant(Entities.Pumpkin)

            move(North)
        move(East)
```

### Improvements
- Introduced can_harvest() logic  
- Reactive farming instead of constant overwriting  
- Cleaner structure  

### Issues
- Still tightly coupled logic  
- Movement + logic mixed  
- No reusable components

## V3 - SmartPlant Modular System

### smartplant module

```python
def sp(entity):
    if entity == Entities.Carrot or entity == Entities.Pumpkin:
        if get_ground_type() == Grounds.Grassland:
            till()
        plant(entity)
    else:
        plant(entity)
```

### Main loop

```python
import smartplant

maxP = get_world_size() * get_world_size()
currentP = 0

while True:
    currentP = 0

    for x in range(get_world_size()):
        for y in range(get_world_size()):

            if can_harvest():
                if get_entity_type() == Entities.Pumpkin:
                    currentP += 1

                if maxP == currentP:
                    harvest()

            smartplant.sp(Entities.Pumpkin)

            move(North)
        move(East)
```

# Key Discovery - System Limitation

A major realization today was that the API does NOT provide any way to detect pumpkin size or “mega pumpkins”.

## Confirmed

- ❌ No Entities.Giant_Pumpkin  
- ❌ No measure() function  
- ❌ No inspect/size API  
- ❌ No growth-stage visibility  

## Conclusion

All pumpkins behave the same from the system perspective.

**Only meaningful signal = can_harvest()**
